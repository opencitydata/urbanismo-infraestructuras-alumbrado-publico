Vocabulary for the representation of the Public lighting: Description back to ToC

The figure from below shows the classes and properties from the vocabulary of the public lighting. The objective of this vocabulary is to portray the parts that form a streetlight, it´s caracteristics and properties.

Now we are going to split the vocabulary in patterns.

3.1 Geo pattern Back to índice

The Equipment class is a subclass of geosparql:Feature class that belongs to the geolocation pattern.

The location pattern is formed by the geo and geosparql vocabularies. To obtain the geolaction we use the object property has geometry that is related with the geosparql:Geometry class. The geosparql:Point class is a subclass of the Geometry class and has the attributes Longitude and Latitude which refer to the coordinate x and coordinate y from the streetlight.

An example of geolocation is shown below with the attributes Latitude and Longitude that belong to the Point class. 

 
    
      <http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/alumbrado-publico#Farola> a esalumb:Streetlight;
      geo:Lat "4480670.67020000000"^^xsd:double;
      geo:Lon "445648.48120000000"^^xsd:double;

3.2 Equipment pattern Back to índice

The equipment pattern is formed by the esequip vocabulary.To obtain the equipment we have the esalumb:Streetlight class that is a subclass from the Equipment class that is used in the municipal equipments that defines the following attributes startingDate which refers to the date when the streetlight was installed, endingDate which refers to the date when the streetlight was desinstalled, id which refers to the id that has every streetlight.

An example with attributes from the Equipment class and from the Streetlight class is shown below.

In this case from the class that represent the municipal equipment we have the identifier that is unique for every streetlight. From the streetlight class we have three attributes: the maximun heigh of the streetlight, the width from one luminary to the opposite one and the control cabinet where the streetlights are controlled.

 
    
      <http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/alumbrado-publico#Farola> a esalumb:Farola;
      esequip:id "1"^^xsd:string;
      esalumb:width "3"^^xsd:string;
      esalumb:heigh "6"^^xsd:string;
      esalumb:controlCabinet "Caseta de Camineros "^^xsd:string;

3.3 Pattern for the location using the administratuve territory and with the schema Back to índice

The pattern of the administrative teritory is formed by the esdir vocabulary and the schema pattern is formed by the schema vocabulary.

The esalumb:Streetlight class is related with the DireccionPostal class with the address object property.

The DireccionPostal class has two properties neighborhood and district, defined by the esadm vocabulary, that are related with the Neighborhood and District classes that refer to the neighborhoodand to the district where the streetlight is located.

The DireccionPostal class is a subclass of the PostalAddress class defined by the schema vocabulary and it has as attributes streetAddress that refers to the complete name of the street including the number and the venue type where the streetlight is located, addressCountry refers to the country where the streetlight is located and addressLocality that refers to the locality where the streetlight is located.

An example with the attributes from the DireccionPostal, PostalAddress, Neighborhood and District classes is shown below.

The neighborhood class represents the neighborhood where the streetlight is located. The district class represents the district where the stretlight is located. The PostalAddress class has three attributes: the first one streetAddres refers to the complete name of the street including the number and the venue type where the streetlight is located, the second attribute addressLocality is refered to the locality where the streetlight is located andthe last attribute addressCountry is refered to the country where the streetlight is located.

 
    
      <http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/alumbrado-publico#Farola> a esalumb:Farola;
      esalumb:code "11116178"^^xsd:string;
      schema:streetAddress "CALLE DE GOMEZNARRO 54"^^xsd:string;
      schema:locality "Madrid"^^xsd:string;
      schema:country "España"^^xsd string;
      esadm:district <http://vocab.linkeddata.es/madrid/recurso/sector-publico/territorio/distrito/Hortaleza>;
      esadm:neighborhood <http://vocab.linkeddata.es/madrid/recurso/sector-publico/territorio/barrio/3>;

3.4 Description of the streetlight components Back to índice

The esalumb:Streetlight class is related with the parts that make up the streetligth with the properties box, pole and luminary. The lamp is related to the luminary using the hasLamp property.

We can also see that the classes have skos lists, the Box class has the skos list that represents the different types of cables that a streetlight can have cableType, the Pole class has a skos list that represents the diferent types of holders that a streetlight can have holderType, the Luminary class has a skos list that represents the the different types of luminaries that a streetlight can have luminaryType and lastly the Lamp class has a skos list that represents the different types of lamps that a streetlight can have lampType.

We have an example with the use of the skos lists and the Luminary class.

We have a holderType skos list that represents the diferent types of holders that a streetlight can have, another skos list for the lampType that represents the different types of lamps that a streetlight can have and the last list is for the luminaryType that represents the the different types of luminaries that a streetlight can have. We also have an attribute that represents the number of luminaries that a streetlight can have.

 
    
      <http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/alumbrado-publico#Farola> a esalumb:Farola;
      esalumb:holderType <http://vocab.linkeddata.es/datosabiertos/kos/urbanismo-infraestructuras/alumbrado-publico/tipo-soporte/poste>;
      esalumb:lampType <http://vocab.linkeddata.es/datosabiertos/kos/urbanismo-infraestructuras/alumbrado-publico/tipo-lampara/led>;
      esalumb:luminaryType <http://vocab.linkeddata.es/datosabiertos/kos/urbanismo-infraestructuras/alumbrado-publico/tipo-luminaria/proyector>;
      esalumb:numberOfLuminaries "1"^^xsd:string;

3.5 Description for the streetlight skos lists Back to índice

The esalumb:Streetlight class is related to three types of skos lists, the first one powerType that refers to the different types of sources a streetlight can have, the second one iluminationType tht refers to the different types of ilumination a strretlight can have and lastly lightingType that refers to the diferents type of lighting that a streetlight can have.

In this example we can see how the three skos lists that the Streetlight class has.

      esalumb:code "2023"^^xsd:string;
      esalumb:powerType <http://vocab.linkeddata.es/datosabiertos/kos/urbanismo-infraestructuras/alumbrado-publico/tipo-alimentacion/subterranea>;

3.6 Examples Go back índice

With the goal of making easier to understand the use of some of these classes and properties of this vocabulary we are going to share an example.

Firstly we can see the description of tha class Streetlight esalumb:Streetlight that collects the caracteristics that cannot be found in other ontologies but are considered necesary for a correct deployment. The data property that has this class in this case id the id that is shared between the streetlights from the same street esalumb:id.

The class esalumb:Lamp has a skos list with the lamp type that is used skos-esalumb:lampType and an atribute that represents the bulb model.

Besides the class esalumb:Streetlight we reuse other classes:

  • For the representation of the points where the streetlights are located we use the atributes longitude and latitude, we have reused the classes geosparql:Feature, geosparql:SpatialObject, geosparql:Geometry and geosparql:Point from the vocabulary geosparql following the "Location pattern" from the figure 1.
  • For the representation of the territories in terms of the neighborhood and distric we have reused the vocabularies esdir:DireccionPostal and esadm following the "Administrative territory" from the figure 1
  • For the representation of the location we have reused the class schema:PostalAddress fro the vocabulary schema following the "schema" from figure 1 

 
    
      http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/alumbrado-publico#Farola" a esalumb:Farola;
      esalumb "20040016"^^xsd:string;
      schema:streetAddress "CALLE DE LAS ERAS 3"^^xsd:string;
      schema:locality "Madrid"^^xsd:string;
      schema:country "España"^^xsd string;
      esadm:district <http://vocab.linkeddata.es/madrid/recurso/sector-publico/territorio/distrito/Hortaleza>;
      esadm:neighborhood <http://vocab.linkeddata.es/madrid/recurso/sector-publico/territorio/barrio/4>;
      esalumb:lampType <http://vocab.linkeddata.es/datosabiertos/kos/urbanismo-infraestructuras/alumbrado-publico/tipo-lampara/descargaɚ;
