Vocabulary for the representation of the Public lighting: Description back to ToC

The figure from below shows the classes and properties from the vocabulary of the public lighting. The objective of this vocabulary is to portray the parts that form a streetlight, it´s caracteristics and properties.

3.1 Ejemplos de uso Go back índice

With the goal of making easier to understand the use of some of these classes and properties of this vocabulary some examples are going to be shown.

First we can see the representaion for the class esalumb:Streetlight that colects the caracetristics that cannot be foound in other ontologies but are neccesary.

The class esalumb:Streetlight), has as data three types of skos lists that identify the type of power (skos-esalumb:powerType), the type of ilumination (skos-esalumb:iluminationType) and the type of lighting (skos-esalumb:lightingType) that the streetlights have; the location (esalumb:Location); the box within the streetlight (esalumb:Box); the pole (esalumb:Pole); the luminary (esalumb:Luminary).

The properties that has this class are:the control cabinet that has a group of streetlights (esalumb:controlCabinet); the owner (esalumb:owner); the streetlight control method (esalumb:controlMethod); the heigh (esalumb:heigh); the width (esalumb:width); and the working hours (esalumb:workingHours).

Also, from the class esalumb:Streetlight are created two subclasees, the first one is equipment (esequip:Equipment) that is the reused class from the municipal equipment vocabulary; the second suclass is about the location pattern that provides the coordinates of the streetlight (geosparql:Feature) that is a reused class from the vocabulary ().
